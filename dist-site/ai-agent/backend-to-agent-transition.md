## 转型 Agent 的 5 个关键点

| # | 方向 | 说明 |
|---|------|------|
| ① | **深度学习 & Transformer 大模型基础** | 找机器学习/Transformer网课（吴恩达/李沐），不需要多深，有基础即可 |
| ② | **后端基础** | 面试中后端知识很重要，大多数面试官都会问。有前端基础也行 |
| ③ | **Agent 相关知识** | 啃 Claude Code 源码（Harness、上下文压缩、长短期记忆、沙箱隔离、多Agent机制） |
| ④ | **Agent 项目** | 一定要吃透，不然是给自己挖坑 |
| ⑤ | **算法题** | 至少 Hot 100 得掌握（虽有大化趋势） |

## 面试官原话（印象深刻）

> "我理解现在校招生、实习生的学习链路和我们不一样，你现在看到的 Agent 相关岗位的面试官，绝大部分都是后端转来的，我也好，你后面可能遇到的其他面试官也好，都是先学后端，再学 Agent 相关技能。但你们现在可能先学 Agent 技能，再通过它了解后端知识，这也不能说不行，有些人愿意带你上手 Agent 技能的项目，**但是没有前后端工程经验，很多时候 Agent 也做不好。**"

## Agent 开发面试 vs 传统后端面试的差异

| 维度 | 传统后端 | Agent 开发 |
|------|---------|-----------|
| 八股文 | ✅ | ✅ |
| 算法题 | ✅ | ✅（弱化趋势） |
| 项目经验 | ✅ | ✅ |
| **系统设计** | 部分考察 | **着重考察** |
| **产品思维** | 较少 | **着重考察** |
| **AI Coding 能力** | 无 | **着重考察** |

### AI Coding 能力 = 全链路闭环

> 给一个需求，自己 **设计 → 开发 → 测试** 的完整闭环

---

## 行业趋势判断

- Agent 开发面试与传统后端面试 **很不一样**
- 校招生/实习生学习链路变化：先学 Agent 技能 → 再补后端知识
- AI 大趋势下有 **全栈化趋势**
- 没有前后端工程经验，Agent 也做不好
- 需要不断学习，没有捷径

---

## 对 Java 开发者的启示

1. **后端基础是核心优势**: 面试官都是后端出身，认可后端基础的价值
2. **补 AI 基础即可入门**: 不需要深度学习算法，有 Transformer 基础就够
3. **Claude Code 源码是最佳教材**: 涵盖 Agent 核心机制（Harness、上下文压缩、记忆、沙箱、多Agent）
4. **项目必须吃透**: 做中学，动手做才能理解深
5. **系统设计 + 产品思维**: 这是 Agent 岗区别于传统后端的核心能力
6. **AI Coding 全链路**: 善用 AI 工具独立完成设计→开发→测试闭环
