# 字节跳动 AI Agent 后端一面面经

> 来源：小红书 | 提取日期：2026-05-22

## 面试问题

1. **自我介绍**

2. **项目中一共用了几个模型？分别是什么？**
   - embedding、路由、LLM、Rerank 各一个

3. **简单讲讲 Agent 项目？**
   - 基于大模型实现 ReAct 模式下的自主规划能力
   - 主要解决长线任务、学习检索和记忆规划

4. **你的 RAG 是用什么技术实现的？**
   - 主要就是讲 RAG 的整体流程和常见技术点

5. **算法题：Leetcode 143，重排链表**

6. **短期记忆的具体实现方式是什么？**
   - 设置一个上下文窗口上限
   - 快到上限时让 Agent 做总结，同时保留最近 2 到 3 轮对话

7. **最近了解的 AI 内容？**
   - OpenClaw、Claude Code 源码

8. **如何减少工具过多带来的 Token 消耗？**
   - 通过意图识别和渐进式披露，只暴露当前任务真正需要的工具

9. **如何判断向量相似度？**
   - 使用余弦相似度

10. **每轮对话都要注入记忆吗？长期记忆是同时注入吗？**
    - 不是，需要根据场景按需召回

11. **BGE 模型的结构是什么？**

12. **你平时调试代码怎么调试？**
    - 一开始答了断点和 AI，面试官提示其实最简单的方式是打印日志

13. **如果对话轮次过多，你怎么优化？**
    - 保留 System Prompt，确保角色不丢
    - 保留最近 2 到 3 轮对话，让模型聚焦当前任务
    - 更久远的内容做总结压缩

14. **RAG 项目里 400 万文字是怎么分块、索引和 embedding 的？**

15. **长期记忆可以按需检索召回，具体什么情况下需要检索？**
    - 可以通过意图路由判断

16. **如何控制模型幻觉？**
    - RAG 召回准确性
    - 引用约束
    - 回答格式约束
    - 检索审计

17. **什么叫"快到上下文上限"？对话内容是怎么叠加的？**
    - 每一轮用户 query 和模型 response 都会进入上下文，并在下一轮一起发送给模型

18. **图片和表格的信息丢失了怎么办？**

19. **如何观察模型召回了哪些 chunk？**
    - 在检索阶段做审计和日志记录

## 考察重点

- Agent 核心机制：ReAct、RAG、Memory
- 工程落地细节：Token 优化、上下文管理、幻觉控制
- 算法能力：链表重排
- 模型理解：BGE、Embedding 技术选型
