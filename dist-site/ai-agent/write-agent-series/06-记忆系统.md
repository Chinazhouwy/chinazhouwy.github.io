当然，这只是一个简单的记忆系统，实际应用中，AI 的上下文是有限制的，Agent 关闭后，上下文就会被清空。

这个时候，我们就要考虑把记忆存储起来，在每次新的聊天开始时，根据需要把存储的记忆，添加到 Agent 上下文即可。

存什么？怎么存？

让我们一起看看 OpenClaw 和 Hermes 是怎么存的，存了什么。

OpenClaw 会把当天所有的会话记录存在一个 Markdown 文件中，认为长期有用的信息，专门用一个 MEMORY.md 文件来存储。

在需要使用时，通过搜索来加载进 Agent 上下文。

Hermes 也类似，记忆由两个小文件组成：MEMORY.md（2200 字符）和 USER.md（1375 字符）。

Hermes 会严格限制记忆文件大小，超过大小后，会自动提取重要的信息存下来。每次会话开始时，把 MEMORY.md 和 USER.md 加载进 Agent 上下文即可。

接下来我们给我们的 Agent 系统添加一个记忆系统。
