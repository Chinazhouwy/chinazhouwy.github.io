《从零开始写 Agent》系列第九篇，是时候让 Agent 替我们干点活了。

说真的，现在好多工作都是在浏览器里进行的。

今天我们就来实现一个让 Agent 可以自动化操作浏览器的功能。

先看一下最终效果。

回到代码这块。

操作浏览器的工具调用的是 agent-browser 的 CLI 工具。

我觉得这个工具特别有意思。agent-browser 是 Vercel Labs 开源的，专为 AI Agent 设计的浏览器自动化 CLI 工具。

它的核心目标是，让 LLM 能以极低的 Token 成本，稳定地操作浏览器。

怎么说呢，传统的浏览器自动化工具，比如 Playwright 或者 Selenium，都太复杂了。给 Agent 用的话，Token 成本太高，而且容易出问题。

这个工具不一样，专门为 Agent 优化过。

下面是给 Agent 增加的操作浏览器工具的完整代码。

说真的，到这里我们的 Agent 就具备了自动化操作浏览器的能力。

有了这个能力，就打开了很多想象空间。

现在大量业务系统都跑在浏览器里，运营配置后台，工单处理系统，发布运维平台。

如果能让这些系统自动运转，对提效和智能化运营的价值不言而喻。

能做什么呢？

旅游预订和价格监控。

电商价格对比与库存检查。

动态网站结构化提取。

具备视觉感知的 UI 测试与验证。

网站监控与告警。

跨多步骤流程的智能表单填写。

怎么说呢，这就像是给 Agent 装上了一双能操作浏览器的手。

以前我们觉得很多自动化的事情很复杂，现在有了这个工具，好像一切都变得简单了。

我是真的觉得这是一个特别有意思的方向。

希望能给你带来一点启发。
