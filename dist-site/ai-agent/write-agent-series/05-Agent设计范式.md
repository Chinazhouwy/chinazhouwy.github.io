稍微了解过一点 Agent 知识的人，可能都听过 Agent 设计范式。

目前主流有三种设计范式：

1. ReAct 设计范式 - 边想边做

2. Plan-and-Execute - 想好了再做

3. Reflection 设计范式 - 做完了再反思

既然 Agent 有不同的设计范式，那必然涉及到如何选择的问题。

说白了其实就是选择如何做事的方式。

大多数人做事肯定都是边想边做的，因为这样最简单。

对于简单的任务来说，边想边做的方式是最合适的。

但是对于长时间任务来说，边想边做的方式可能就容易走偏

工作几年的人，可能就会发现，做事情之前，最好先想一下，做好计划

这样可以确保做事情的时候，不会走偏，也不会忘记重要的事情。

追求更高质量的人，可能会在做完了事情之后再回过头来检查一下，经过几轮的打磨，这样就可以确保事情做的更好。

从我们人做事的方式上可以看出，其实选择 Agent 设计范式，就是选择希望 AI 如何做事。

这个时候你可能会说，当然是有计划做反思能把事情做得更好。

那 ReAct 设计范式，是不是就没有存在的必要了？

那显然不是，就像我们即使工作了很多年，也并不是每件事都需要有计划，也并不是每件事都需要有反思的。

很多事情我们一开始都是边想边做的。

在选择 Agent 设计范式的时候是同样的道理。

具体怎么选呢？其实很简单，就看你要做什么事：

• 
简单的小事
：用 ReAct，边想边做，灵活快速

• 
复杂的任务
：用 Plan-and-Execute，先做好计划再执行

• 
高质量要求
：加上 Reflection，执行完再反思打磨

其实从我们自己做事的方式，就能很好地理解这三种设计范式。

你可以回忆一下自己：

• 今天下午想喝杯咖啡，你会先做个计划吗？不会的，直接 ReAct 就去了

• 要做个年度汇报，你可能会先规划一下大纲，再逐步填内容

• 写完稿子，你会再读一遍，改改错别字，调整一下表达

对，就是这么简单。

选择 Agent 设计范式，本质上就是让 AI 用你想要的方式来做事。

没有最好的范式，只有最合适的范式。

其实还有一个问题，就是这些设计范式在我们的 Agent 的实现中是如何体现的呢？

后面的文章我会拆解现在主流的 Agent 实现中，这些设计范式的具体实现。
