# 网易一面（技术+Leader）AI Agent 面经

> 来源：用户分享 | 提取日期：2026-06-15 | 面试轮次：一面（技术+Leader）

## 面试问题

**Q1: 自我介绍**

**Q2: 说一下第一个项目的背景和难点**

**Q3: 多Agent如何设计的，为什么是三层架构?**
- 考点：多Agent协作模式选择、分层架构的理由
- 需要能解释清楚三层各自的职责、为什么不能两层或四层

**Q4: 如果有个Agent执行失败了，处理逻辑是什么?**
- 考点：Agent容错机制、重试策略、降级方案
- 需要说明失败检测、重试策略、告警机制、兜底方案

**Q5: 人怎么保证任务拆得准确的，防止幻觉的机制有哪些?**
- 考点：Agent任务分解准确性、防幻觉
- 涉及：Plan验证、执行结果校验、约束条件、反馈闭环

**Q6: 异步Channel机制是什么，如何实现的?**
- 考点：异步通信机制、Agent间消息传递
- 可能涉及：消息队列、事件驱动、回调机制

**Q7: DAG如何知道依赖关系，怎么判断该执行还是不执行，DAG会不会死循环?**
- 考点：DAG编排、依赖解析、循环检测
- 核心：拓扑排序、环检测算法、条件执行判断

**Q8: 举一个实际业务场景Agent真实提效的例子**
- 考点：Agent落地价值、量化指标
- 需要具体数据：效率提升比例、节省人力、准确率等

**Q9: 目前多少人在使用?**
- 考点：产品规模、用户覆盖

**Q10: 由于和业务结合，如何保证安全审计的?**
- 考点：Agent安全设计、审计日志、权限控制
- 涉及：操作日志、敏感操作审批、数据脱敏

**Q11: 沙箱设计的规则是哪些?**
- 考点：代码执行沙箱、安全隔离
- 可能涉及：资源限制、权限隔离、超时控制、输出过滤

**Q12: 说一下问答助手架构设计**
- 考点：RAG系统设计、检索增强生成架构
- 核心：检索策略、Prompt工程、上下文管理、答案生成

**Q13: A2A在你的项目当中是怎么使用的，其他业务Agent怎么接入?**
- 考点：Agent-to-Agent协议、多Agent接入标准
- 涉及：A2A协议规范、接口定义、发现机制、注册中心

**Q14: 文档权限是怎么控制的?**
- 考点：细粒度权限设计
- 可能涉及：RBAC、文档级权限、操作权限

**Q15: 文档更新机制?**
- 考点：增量更新、版本控制、并发处理

**Q16: 知识每次检索都是从头开始吗，了解LLM Wiki吗?**
- 考点：RAG检索优化、知识组织方式
- LLM Wiki：知识图谱化组织、分层索引、增量更新

**Q17: 反问**
