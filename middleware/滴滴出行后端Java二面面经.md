## 三、高频原题

### 1. 项目终极深挖（二面占分最重）

- 画项目完整架构图，从用户请求入口到数据库全链路逐段拆解
- 项目峰值 QPS、日均订单量、单表数据量、接口 P99 延迟、并发峰值
- 你独立负责核心模块，在整个业务链路的定位、核心流程、技术设计
- 项目最大技术难点完整复盘：发现问题 → 根因分析 → 多方案对比 → 落地实现 → 效果数据 → 复盘优化
- 技术选型 Trade-off：为什么用 Redis/RocketMQ/分库分表/微服务？不用其他方案的劣势？
- 接口 P99 延迟高、超时、抖动怎么排查？用过链路追踪、监控告警吗？
- 做过哪些性能优化？优化前后 QPS、RT、CPU、GC、内存对比
- 高并发下单场景怎么做幂等、防重复下单？具体落地实现
- 订单库存/资源占用怎么防超卖、保证状态一致性？
- 分布式事务用什么方案？可靠消息/TCC/SAGA？优缺点、踩过什么坑？
- 缓存和数据库双写一致性几种方案对比，你项目用哪种、怎么解决延迟不一致
- 流量突增10倍，系统最先瓶颈在哪？怎么限流、降级、扩容、业务兜底
- 微服务调用超时、重试、熔断，怎么防止服务雪崩？滴滴内部怎么治理？
- 如果从零重构这个项目，你会改掉哪3个设计缺陷？

### 2. Java 集合进阶（深挖源码）

- HashMap 1.7/1.8 底层结构、put 流程、扩容、树化完整逻辑
- 链表转红黑树阈值为什么是 8、退化链表为什么是 6？
- HashMap 并发安全问题，1.8 还会出现循环链表吗？
- ConcurrentHashMap 1.8 CAS+Synchronized 锁粒度，为什么放弃分段锁？
- CopyOnWriteArrayList 原理、读写分离、适用场景、缺点
- LinkedHashMap 实现 LRU 原理，手写简易 LRU 思路

### 3. Java 并发（滴滴二面重灾区）

- synchronized 完整锁升级流程：偏向锁→轻量级锁→重量级锁
- 偏向锁什么时候撤销？重偏向触发条件？
- synchronized 和 ReentrantLock 底层、性能、功能、场景全方位对比
- AQS 核心源码：双向同步队列、独占/共享模式、Condition 等待队列
- ReentrantLock 公平/非公平、可重入底层怎么实现
- CountDownLatch、CyclicBarrier、Semaphore 底层区别和出行业务实际场景
- CAS 底层 Unsafe 原理、优缺点、ABA 问题及工程级完整解决方案
- volatile 内存屏障、禁止指令重排序原理，DCL 单例为什么必须加 volatile
- 线程池七大参数逐字详解、任务提交执行全流程
- 核心线程数科学配置公式，CPU 密集/IO 密集怎么选
- 线程池线上经典坑：任务异常吞没、队列无限堆积、空闲线程回收
- ThreadLocal 底层哈希冲突、内存泄漏根因、和线程池混用风险、业务正确用法

### 4. JVM 深度（偏线上调优 + 故障排查）

- JVM 运行时数据区 5 个区域，各自作用、分别会发生什么 OOM
- 对象创建全过程、对象头 MarkWord 结构、锁状态存在在哪里
- GC 三大算法优缺点、分代回收设计思想
- CMS 四步回收流程、优缺点、内存碎片、并发失败问题
- G1 分区模型、可预测停顿、Mixed GC 机制
- ZGC 低延迟核心原理、颜色指针、几乎无 STW、不分代
- CMS/G1/ZGC 三者选型对比，出行订单低延迟业务怎么选？
- 双亲委派模型作用、哪些场景需要打破双亲委派？
- 线上 CPU 飙高、频繁 FullGC、OOM 完整排查定位流程
- 逃逸分析、栈上分配、标量替换原理，对高并发接口性能提升作用

- 幻读根本成因、间隙锁 + 临键锁如何解决幻读？
- 行锁、表锁、意向锁、间隙锁、临键锁各自触发场景
