## 一、技术深度：懂底层，能调优

### 1.1 面试风格

- **不考八股**: 不问概念，问实战
- **必答题**: 系统容量边界在哪？下游大规模超时，系统怎么崩？

### 1.2 基础素养

| 技术领域 | 考察内容 | 实战问题示例 |
|---------|---------|-------------|
| JVM | GC调优 | 如何分析Full GC频繁？参数如何调优？ |
| 多线程并发 | 线程池、锁机制 | 联合索引为什么失效？ |
| 网络IO | NIO/AIO、连接池 | 高并发下连接池如何配置？ |
| MySQL | 索引与锁 | 慢查询如何优化？ |
| Redis | 底层数据结构 | 缓存穿透/击穿/雪崩怎么解决？ |

### 1.3 自研中间件

腾讯内部中间件需要了解：

| 中间件 | 用途 | 面试考点 |
|--------|------|---------|
| PhxPaxos | 分布式一致性 | Paxos算法、分布式共识 |
| TDMQ | 消息队列 | 消息可靠性、堆积处理 |
| TDSQL | 分布式数据库 | 分库分表、分布式事务 |
| Zhiyan | 日志排查 | 日志分析、故障定位 |

### 1.4 成本意识

**场景题**: 二级缓存 vs 纯Redis

需要从以下维度量化对比：
- **一致性**: 二级缓存一致性如何保证？
- **维护成本**: 复杂度、运维难度
- **不可用面**: 故障影响范围

> ⚠️ 面试要点: 技术选型不能只说优点，要从成本、风险、收益多维度分析。

public SentinelConfig sentinelConfig() {

@Configuration
public class ThreadPoolConfig {
    @Bean("criticalExecutor")
    public ThreadPoolExecutor criticalExecutor() {
        return new ThreadPoolExecutor(
            20, 50, 60, TimeUnit.SECONDS,
            new ArrayBlockingQueue<>(500),
            new ThreadPoolExecutor.CallerRunsPolicy()
        );
    }
}

@Service
public class DegradationService {
    @Value("${degradation.order.enabled:false}")
    private boolean degradationEnabled;
    
    public Order createOrder(OrderDTO dto) {
        if (degradationEnabled) {
            // 降级: 返回默认值或缓存数据

        }
        // 正常流程

    }
}

### 2.5 架构取舍

**引入MQ的闭环要求**:

| 问题 | 解决方案 |
|------|---------|
| 防丢 | 生产确认（Confirm机制） |
| 防重 | 幂等设计（唯一索引/Redis SET NX） |
| 防积压 | 动态扩容（消费者水平扩展） |

**双写一致性场景**:
- "先删缓存后更新库"的风险：并发场景下可能读到旧数据
- 推荐方案：先更新数据库，再删除缓存，配合延迟双删或订阅Binlog

> ⚠️ 工程踩坑: 缓存和数据库的双写一致性是经典难题。推荐 "先更新数据库，再删除缓存" 的方案，配合延迟双删或订阅 Binlog 异步删除。

// 1. 限流 + 队列削峰
@Service
public class RedPacketService {
    
    private final RateLimiter rateLimiter = RateLimiter.create(100000.0);
    
    @Autowired private RocketMQTemplate rocketMQTemplate;
    
    public boolean grabRedPacket(Long packetId, Long userId) {
        // 限流
        if (!rateLimiter.tryAcquire(100, TimeUnit.MILLISECONDS)) {
            return false;
        }
        
        // Redis预扣减
        String key = "redpacket:stock:" + packetId;
        Long stock = redisTemplate.opsForValue().decrement(key);
        if (stock < 0) {
            redisTemplate.opsForValue().increment(key);
            return false;
        }
        
        // 异步创建记录
        rocketMQTemplate.convertAndSend("redpacket.grab", 
            JSON.toJSONString(new GrabRecord(packetId, userId)));
        return true;
    }
}

// 2. 最终一致性
@Component
public class RedPacketConsumer {
    
    @RocketMQMessageListener(topic = "redpacket.grab", consumerGroup = "redpacket-group")
    public class GrabConsumer implements RocketMQListener<String> {
        @Override
        public void onMessage(String message) {
            GrabRecord record = JSON.parseObject(message, GrabRecord.class);
            // 幂等处理
            if (grabRecordMapper.exists(record.getPacketId(), record.getUserId())) {
                return;
            }
            // 创建记录
            grabRecordMapper.insert(record);
            // 对账补偿
            reconcileService.checkAndFix(record);
        }
    }
}

// 3. 降级预案
@Service
public class RedPacketDegradation {
    
    @Value("${redpacket.degradation.enabled:false}")
    private boolean degradationEnabled;
    
    public RedPacketResult grab(RedPacketDTO dto) {
        if (degradationEnabled) {
            // 降级: 返回排队结果
            return RedPacketResult.queuing();
        }
        return doGrab(dto);
    }
}
