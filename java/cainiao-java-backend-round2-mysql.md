## 面试题

### 1. InnoDB 和 MyISAM 的区别

**答题思路：**
- InnoDB：默认引擎，支持事务、行锁、外键、MVCC
- MyISAM：不支持事务、表锁、查询快，适合静态数据场景
- 生产环境基本不用 MyISAM

### 2. MySQL 索引原理，B+树为什么优于B树？

**答题思路：**
- B+树所有数据存在叶子节点，叶子节点形成有序链表
- 非叶子节点只存索引，磁盘IO次数更少
- 查询稳定（每次到叶子层）、范围查询高效（链表遍历）

### 3. 哪些情况会导致索引失效？

- `LIKE '%xxx'` 左模糊查询
- 索引列参与运算 `WHERE id+1=5`
- 使用函数 `WHERE DATE(create_time)=...`
- 隐式类型转换 `WHERE phone=123`（phone是varchar）
- `OR` 连接无索引字段
- 违反最左前缀原则

### 4. 四大事务隔离级别

| 级别 | 脏读 | 不可重复读 | 幻读 |
|------|------|-----------|------|
| 读未提交 | ✅ | ✅ | ✅ |
| 读已提交 | ❌ | ✅ | ✅ |
| 可重复读（MySQL默认） | ❌ | ❌ | ✅ |
| 串行化 | ❌ | ❌ | ❌ |

### 5. MVCC 原理

- **多版本并发控制**，通过 undo log 版本链 + Read View 实现
- 无锁读写，提升并发性能
- 可重复读隔离级别的核心实现原理
- Read View 四个字段：`m_ids`（活跃事务ID列表）、`min_trx_id`、`max_trx_id`、`creator_trx_id`

### 6. 慢SQL如何排查和优化？

- 开启慢查询日志 `SET long_query_time=1`
- `EXPLAIN` 分析执行计划（type、rows、Extra）
- 优化索引（覆盖索引、联合索引最左前缀）
- 避免全表扫描
- 拆分大SQL、分页优化（子查询延迟关联）
- 减少关联查询

### 7. 项目中有没有做过SQL优化？

结合自己项目举例：新增索引、优化模糊查询、拆分复杂SQL等实操案例。
