## 2. 业务架构设计大题（饿了么真实场景）

### Q1：设计骑手 GPS 实时定位上报系统，要求用户端 500ms 内响应，如何实现？

**答题要点：**
- 高频上报数据降噪
- 增量上报
- Redis 缓存骑手位置
- 分片存储
- 读写分离
- 本地缓存兜底
- 异步持久化
- 前端组存优化响应速度

### Q2：订单数据十亿级，如何设计分片、归档、查询架构？

**答题要点：**
- 按时间 + 用户 ID 复合分片
- 冷热数据分离
- 历史订单归档至 ES / 冷库
- 实时订单查热库
- 分页查询优化
- 避免跨分片查询

## 3. 框架源码 & 实战调优

### Q1：Spring AOP 原理、事务传播机制，事务失效场景有哪些？

**答题要点：**
- 动态代理实现 AOP
- 7 种事务传播机制
- 失效场景：
  - 非 public 方法
  - 内部调用（自调用）
  - try-catch 吞异常
  - 多线程事务
  - 数据库不支持事务

### Q2：线上接口响应变慢，完整排查思路？

**答题要点：**
- 先看监控（QPS、耗时、报错率）
- 排查 GC
- 数据库慢 SQL
- 缓存命中率
- 网络延迟
- 线程阻塞
- 第三方接口耗时
