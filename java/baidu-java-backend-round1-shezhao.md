## 2、项目深挖（核心重点）

### Q1：介绍你最核心的一个后端项目，你的核心职责是什么？

**追问1**：项目中遇到的最大技术难点是什么？如何排查、如何解决？

**追问2**：项目中做过哪些性能优化？优化前后数据对比？

**追问3**：项目接口并发量多少？如何保证高可用？

### Q2：项目中是否用到秒杀/高并发场景？如何解决？

**参考答案**：
- 前端防刷节流，减少无效请求
- 后端基于 **Redis 分布式锁**、**乐观锁** 防止超卖
- **幂等性设计** 避免重复请求
- **队列削峰填谷** 应对瞬时高并发

## 3、分布式基础问答

### Q1：分布式ID生成策略有哪些？各自优缺点？

| 策略 | 优点 | 缺点 |
|------|------|------|
| **雪花算法** | 高并发、有序、无重复 | 依赖机器时间（时钟回拨问题） |
| **UUID** | 实现简单 | 无序、数据库索引效率低 |
| **数据库自增** | 有序、简单 | 并发性能差 |
| **Redis 自增** | 高性能 | 需保证持久化 |

### Q2：分布式负载均衡有哪些方式？

- **服务端负载均衡**：Nginx 轮询、权重、IP 哈希
- **客户端负载均衡**：Ribbon（从注册中心获取服务列表，本地负载均衡）

## 4、基础穿插提问

### Q1：进程和线程的区别？多CPU下多线程的意义？

- **进程**：资源分配最小单位，开销大、隔离性强
- **线程**：CPU 调度最小单位，开销小、共享资源
- 多CPU下多线程：充分利用多核资源，提升任务并行处理效率

### Q2：死锁产生的四个必要条件？如何避免死锁？

**四个必要条件**：
1. 互斥条件
2. 持有并等待
3. 不可剥夺
4. 循环等待

**避免方式**：
- 统一锁的获取顺序
- 设置锁超时时间
- 破坏循环等待条件

## 5、手撕代码

**题目**：二叉树层序遍历（简单难度）

```java
// BFS 解法
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;
    
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    
    while (!queue.isEmpty()) {
        int size = queue.size();
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {
            TreeNode node = queue.poll();
            level.add(node.val);
            if (node.left != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        result.add(level);
    }
    return result;
}
```
