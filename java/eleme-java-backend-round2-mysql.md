## 2. MySQL 核心考点

### Q1：MySQL 四大事务隔离级别，分别解决什么问题？默认级别是什么？

**答题要点：**
- 读未提交（RU）、读已提交（RC）、可重复读（RR）、串行化（Serializable）
- 分别解决脏读、不可重复读、幻读问题
- InnoDB 默认 **可重复读（RR）**，通过 **间隙锁** 解决大部分幻读

### Q2：B+ 树结构特点，MySQL 为什么选用 B+ 树而不是 B 树、红黑树？

**答题要点：**
- B+ 树非叶子节点只存索引，叶子节点存全部数据且有序链表串联
- 磁盘 IO 更少、范围查询高效、层高更低
- 红黑树层高更高、不适合磁盘存储
- B 树范围查询效率差

### Q3：联合索引最左匹配原则，失效场景有哪些？

**答题要点：**
- 违背最左前缀
- 索引列参与运算
- 字段隐式转换
- `LIKE` 左模糊查询
- `OR` 连接无索引字段

### Q4：千万级订单表如何优化？分库分表策略、冷热数据处理？

**答题要点：**
- 水平分表（按用户 ID / 时间分片）
- 冷热分离（历史订单归档至冷表）
- 建立合适联合索引
- 避免大事务
- 定期清理冗余数据

## 3. 业务 SQL 场景题

**饿了么订单超时未支付自动关闭，如何通过数据库 + 业务实现，保证高效不卡顿？**
