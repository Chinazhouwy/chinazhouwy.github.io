# 顺丰后端Java二面面经

> 来源：小红书 | 时长：30分钟 | 面试风格：以项目为核心，层层追问细节

## 面试流程

项目介绍 → 项目深挖追问 → 框架选型 → 缓存设计 → 网络基础 → 职业规划

## 面试题

### 一、项目深挖

1. 项目中为什么选择 Spring Boot/Spring Cloud 框架？对比其他框架的优势？
2. 项目开发中遇到的最棘手的技术难题是什么？详细说明排查过程、解决方案、复盘优化思路？
3. 回看自己的项目代码，还有哪些可以优化的点（性能、代码规范、架构层面）？
4. 项目中用到了哪些算法或工具类，具体解决了什么问题？

### 二、缓存设计

1. 项目中两级缓存的设计思路是什么？为什么这么设计？
2. 两级缓存如何保证数据一致性？如果业务需要强一致性，该如何改造？
3. Redis 常用数据结构及适用场景（String、Hash、List、ZSet）

### 三、网络基础

1. OSI 七层模型、TCP 三次握手、四次挥手详细过程
2. TCP 和 UDP 的核心区别，各自适用的业务场景？

### 四、职业规划

1. 本科专业偏 Python，为什么求职 Java 后端？
2. 日常的 Java 技术学习方法、知识沉淀方式？

## 面试总结

- 框架选型逻辑考察：为什么选 Spring Boot/Cloud？
- 两级缓存数据一致性是高频场景题
- 网络基础是保底题
