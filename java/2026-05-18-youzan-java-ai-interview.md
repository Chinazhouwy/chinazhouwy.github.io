## 一、并发编程与 JVM

1. **线程池的底层执行机制**：具体流程（核心线程 → 队列 → 新增线程 → 拒绝策略）
2. **队列类型偏好**：如 LinkedBlockingQueue，设不设上限？
3. **线程池队列过大的风险**：内存溢出
4. **如何排查线程池 OOM**
5. **原子性 (Atomicity) 和可见性 (Visibility)** 的概念
6. **HashMap 在 JDK 1.8 中如何保证线程安全**：CAS 和 synchronized 的具体应用场景

7. **HashMap 的数据结构和扩容机制**：链表与红黑树并存、扩容流程

## 二、数据库与索引

8. **MySQL 慢查询分析**：EXPLAIN 的使用

10. **查询结合 ORDER BY 对索引的影响**
11. **InnoDB 索引底层数据结构**：B+树
12. **B+树 vs 红黑树**：减少磁盘 IO
13. **顺序 IO 和随机 IO 的区别**
14. **MVCC（多版本并发控制）工作原理**

## 三、缓存与消息队列

17. **缓存穿透、击穿、雪崩**：概念 + 解决方案（缓存空值、随机过期时间、互斥锁）

## 经验总结

1. **并发要懂全流程**: 线程池执行链路、OOM 排查是高频考点
2. **数据库要懂底层**: B+树 vs 红黑树的对比（磁盘 IO 角度）是经典问题
3. **缓存三板斧**: 穿透/击穿/雪崩的概念 + 解决方案必须熟练
4. **双层缓存一致性**: Caffeine 本地缓存 + 分布式通知（HTTP广播/Kafka）是亮点方案
5. **AI 技术栈前沿**: 除了 RAG，还问了知识图谱、Anti-RAG、ES 倒排索引等
6. **AI 辅助编程**: 面试官关注候选人是否有实际使用 AI 工具编码的经验
7. **HashMap 注意**: 作者提醒 AI 总结的 HashMap 线程安全部分可能有误（HashMap 本身非线程安全，应指 ConcurrentHashMap）
