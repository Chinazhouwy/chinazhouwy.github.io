# 百度后端 Java 终面面经

> 来源：[小红书](http://xhslink.com/o/osKAVrzuBn)（OCR 提取） | 提取日期：2026-05-22
> 侧重高并发 + 系统设计 + 业务理解，偏实战应用
> 面试时长：50 分钟 | 面试官：资深后端架构师 | 面试形式：线下面试

---

## 1. 自我介绍（30 秒）

重点突出高并发、高可用相关项目经验，以及自身在项目中承担的核心职责。

## 2. 技术提问

### (1) 高并发场景处理

**面对每秒 10 万+ 请求的场景，你会从哪些方面进行系统优化？**

从缓存、负载均衡、数据库分库分表、异步处理等维度说明。

**说说你项目中处理高并发的具体实践，比如如何避免缓存穿透/击穿/雪崩。**

### (2) 系统设计

**设计一个短链接生成系统**，要求支持高并发、高可用，说明核心流程、数据存储方案、接口设计，以及如何应对峰值流量。

**设计一个分布式定时任务调度系统**，考虑任务分发、执行、重试机制。

### (3) 中间件应用

**Kafka 的底层架构**，分区和副本的作用，如何保证消息的可靠性传输？

**Spring Cloud 的核心组件**，服务注册与发现、负载均衡、熔断降级的实现原理，你在项目中如何使用这些组件？

### (4) Java 基础

- **HashMap 的底层实现（JDK 1.8）**，哈希冲突的解决方式，红黑树的作用
- **Java 的四种引用类型**，以及各自的应用场景
- **泛型的类型擦除实现**，为什么不能用基本数据类型作为泛型参数？

## 3. 业务相关提问

1. 你了解百度的搜索业务或云业务吗？说说你对这些业务的理解，以及后端 Java 开发在其中的作用。
2. 如果让你参与百度搜索的后端开发，你会关注哪些核心指标？如何优化这些指标？

## 4. 个人相关

1. 说说你最擅长的 Java 技术领域，为什么擅长？做过哪些相关的深入研究或实践？
2. 工作中遇到和同事技术意见不一致的情况，你如何处理？
3. 你有什么想问我的？

## 面试总结

> 侧重实战能力和系统设计思维，会结合百度实际业务场景提问，需要具备高并发场景处理经验和系统设计能力，对百度业务有一定了解会加分。
