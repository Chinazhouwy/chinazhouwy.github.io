上一篇我们探讨了 ChatModel 的多厂商接入。今天我们聊 Tool System——Agent 是怎么通过工具跟外部世界进行交互的。

AgentScope 的工具体系分三层：Tool（工具定义）、Toolkit（注册中心+调度引擎）、ToolGroup（分组管理）。下面我们通过源码拆开来分析。

整体设计思路

先看一个宏观问题：要让 LLM 调用工具，中间要解决几个事情。

工具的描述要能让 LLM 理解——所以需要 JSON Schema

工具的参数要从 LLM 返回的 JSON 反序列化——所以需要带类型的参数声明

多个工具可能由同一段逻辑管理——所以需要分组

工具来源不只 Java 方法，还有外部 MCP 服务器——所以需要统一抽象

AgentScope 的思路是：用 

AgentTool

 接口做统一契约，

@Tool

 注解做最简路径，

ToolBase

 做复杂场景的基类。三个概念捆在一起：

Tool

：实现 

AgentTool

 接口的对象，暴露 name、description、parameters（JSON Schema）

Toolkit

：容器。注册 Tool、MCP 客户端、Skill，统一暴露 JSON Schema 给 LLM，收到调用请求后分派到具体工具

ToolGroup

：命名分组。可以整组激活/停用，模型通过内建的 

reset_tools

 元工具在运行时自己切换

自动调度策略：批次/顺序/并发

这是我觉得 AgentScope 设计得比较讲究的地方。看 

ToolExecutor

 的 

executeAll

 方法：

// 文件：agentscope-core/src/main/java/io/agentscope/core/tool/ToolExecutor.java

Mono<List<ToolResultBlock>>

 executeAll

(

        List<ToolUseBlock> toolCalls,

        boolean

 parallel,

        ExecutionConfig executionConfig,

        Agent agent,

        RuntimeContext agentRuntimeContext)

 {

    if

 (!parallel) {

        // 顺序执行：用 Flux.concat，保持调用顺序

        List<Mono<ToolResultBlock>> monos = toolCalls.stream()

                .map(tc -> executeWithInfrastructure(tc, ...))

                .toList();

        return

 Flux.concat(monos).collectList();

    }

    // 并行模式——分批调度

    List<Flux<ToolResultBlock>> chunks =

 new

 ArrayList

<>();

    List<Mono<ToolResultBlock>> safeBatch =

 new

 ArrayList

<>();

    for

 (ToolUseBlock toolCall : toolCalls) {

        Mono<ToolResultBlock> mono = executeWithInfrastructure(toolCall, ...);

        if

 (isConcurrencySafe(toolCall)) {

            safeBatch.add(mono);

         // 攒一批安全的

        }

 else

 {

            if

 (!safeBatch.isEmpty()) {

                chunks.add(Flux.mergeSequential(safeBatch));

 // 发射安全的

                safeBatch =

 new

 ArrayList

<>();

            }

            chunks.add(mono.flux());

     // 不安全的单独一个 slot

        }

    }

    if

 (!safeBatch.isEmpty()) {

        chunks.add(Flux.mergeSequential(safeBatch));

    }

    return

 Flux.concat(chunks).collectList();

}

这干了啥？在并行模式下，

ToolExecutor

 会把工具调用分成连续的"安全批"——一批里全是 

concurrencySafe

 的工具，它们通过 

Flux.mergeSequential

 并发执行（输出顺序按输入顺序排列）。碰到一个不安全的（比如写文件的工具），先发射当前这批，再单独一个 slot 执行不安全的。

这设计解决了一个实际问题：同一个 Agent 在一次回复里可能调多个工具（比如同时查三个城市的天气，再写一个文件）。天气查询互相不冲突，可以并发；写文件要串行化。框架自动做了这个分批，用户只需要在 

@Tool

 上声明 

concurrencySafe

。

MCP (Model Context Protocol) 集成

MCP 是 Anthropic 推的开放协议，解决的是"工具来源"的问题。你可以跑一个 MCP 服务器（比如文件系统工具、数据库工具、浏览器自动化工具），Agent 通过协议连上去发现和使用这些工具。

AgentScope 对 MCP 的支持体现在三块：

三种传输方式

：StdIO（本地子进程）、SSE（HTTP 长连接）、Streamable HTTP

命名空间隔离

：MCP 工具的完整名称是 

mcp__{server_name}__{tool_name}

，避免跟本地工具重名

权限集成

：声明了 

readOnlyHint

 的工具被权限系统自动放行

看 StdIO 的例子：
