千呼万唤始出来，经历了从2.0.0-RC1~2.0.0-RC5版本的迭代，今天2.0.0正式发布版终于来了，相比之前的 RC 版本，2.0.0 GA版新增了 PostgreSQL 状态存储、mem0 记忆集成、RAG 集成等模块，源码也有细微调整。后面我们就拿2.0.0 GA版继续探讨学习。

聊聊上下文这件事

写 Agent 的都知道，LLM 本身是无状态的。你发一条消息过去，它回一条，下一轮就忘了前面说过什么。所以每个 Agent 框架都得解决同一个问题——上下文怎么存、怎么传、怎么恢复。

AgentScope 2.0.0 的做法和我见过的其他框架不太一样。它不是让每个 Agent 实例自己抱着一个 Session 对象不放，而是把 Agent 设计成了

无状态引擎

。什么意思？一个 Agent 实例可以同时服务上千个用户会话，每个会话的状态独立存储、独立加载，互不干扰。

翻阅源码之后觉得这个设计挺聪明的。它把"不变"和"变"拆开了：

不变的东西

：System Prompt、Model 配置、Toolkit、Middleware 链——这些在 Agent 构建时就定死了，之后不会改

变的东西

：对话历史、当前迭代次数、工具激活状态、计划模式开关——这些按 

(userId, sessionId)

 分槽存放

这种分离带来的好处很直接：你不用为每个用户创建一个 Agent 实例，一个 

HarnessAgent

 就能撑起整个业务。

回过头来看，AgentScope 2.0.0 的上下文管理本质上思路很清晰：
