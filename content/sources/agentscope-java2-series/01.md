最近一直在找一套JAVA能真正落地的 LLM Agent 框架。不是那种调了 API 就完事的玩具框架，而是在生产环境里跑得稳、能水平扩展、出了问题能追回来的东西。LangChain4j 算一个选项，但它偏 Python 那套思路。Spring AI 也在做，但抽象层太薄，Multi-Agent 协作基本靠你自己搭。

AgentScope Java 2.0 是我最近深度体验的一套框架，来自阿里通义实验室。今天这篇是系列第 1 篇，先给大家拆一下框架的全貌：它是什么、能干什么、三个核心抽象怎么设计的，最后带大家搭个环境跑通第一个 Agent。

AgentScope 是什么

AgentScope Java 是阿里通义实验室开源的面向 Java 生态的 Agent 框架，Apache 2.0 协议，目前最新版本是 2.0.0-RC4。跟市面上其他框架最大的区别在于：它不只是把 LLM 调用封装了一下，而是提供一个从

单 Agent 推理

到

企业级分布式部署

的完整套件。

项目地址：https://github.com/agentscope-ai/agentscope-java（4.2k stars，活跃维护中）

项目模块结构

从 GitHub 仓库的目录结构可以看到，它不是一个单体项目，而是 Maven 多模块设计：

agentscope-core

               # 核心抽象：Agent、Msg、Model、Toolkit

agentscope-harness

            # 工程化层：Workspace、Memory、Sandbox、Subagent

agentscope-examples

           # 示例代码

agentscope-extensions/

        # 扩展集成

  agentscope-extensions-model/

       # 模型接入（DashScope、OpenAI、Anthropic 等）

  agentscope-extensions-sandbox/

     # 沙箱执行环境

agentscope-dependencies-bom

   # BOM 依赖管理

agentscope-distribution

       # 分发包

agentscope-core

 是最底层的核心，只负责 ReAct 推理循环和基础抽象。

agentscope-harness

 是 2.0 的新主力，在 core 之上叠加了工作区、长时记忆、上下文压缩、技能仓库等工程化能力。用官方的话说：Harness 是让 Agent 能"stay up"的工程脚手架。

2.0 做了什么升级

我看了官方文档的 What’s AgentScope 2.0 页面，三个方向很明确：

Harness Engineering

：给长时间运行的 Agent 提供工程支撑。包括三层记忆管理（上下文/MEMORY.md/磁盘日志）、自动上下文压缩、子 Agent 编排、计划模式。

企业级分布式部署

：Agent 实例是

无状态的

——任何副本都能用 (userId, sessionId) 接起完整对话。Redis/MySQL 状态存储、沙箱隔离、权限系统三态引擎（allow/approve/deny）。

核心框架重构

：Message 模型简化了，Event 流式化成了内置能力，v1 的 Hook 系统升级成了分层 Middleware。

这些概念后面会逐个展开，今天先聚焦三个核心抽象。
