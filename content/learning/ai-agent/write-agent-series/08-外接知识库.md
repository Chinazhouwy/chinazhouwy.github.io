之前的文章中我们提到上下文工程决定了 AI 所能看到的信息，从而影响 AI 的决策和行为。

要想 Agent 能够完成长时程任务，需要把上下文信息从「一次性内存」变成「可查询的知识库」。

今天我们就来给 Agent 实现这个功能。

要实现这个功能其实也就两步：

一、知识库的构建

二、给 Agent 增加从知识库获取信息的工具

知识库，就是把文档、笔记、经验、资料，整理存起来的「知识仓库」。

聊完定义之后，下面就开始构建知识库。

知识库对外提供两个接口，一个是文档导入，一个是语义搜索，这里我们封装成 CLI 工具。

把文档向量化和提取实体关系都是为了后续 Agent 能搜索到需要的信息。

关键词检索 vs 语义搜索

这里简单介绍下，关键词检索和语义搜索区别

关键词检索

：找长得像的文字 

语义搜索

：找意思像的内容

举个例子你就懂了。

假设你的文档里写了夏天怎么防蚊子的知识

关键词搜索，你搜「

怎么防蚊子

」能搜到，搜「怎么不被蚊子咬」搜不到，搜「灭蚊方法」也搜不到，太死板了。

语义搜索不一样，不管你搜「如何防蚊子」，还是「怎么不被蚊子咬」，还是「灭蚊方法」，它都能找到。

为啥？因为它找的不是长得像的文字，是意思像的内容。

再加上知识图谱，你搜「居家生活技巧」，它能给你推荐「防蚊子」，你搜「深圳的夏天怎么样」，它能告诉你深圳夏天蚊子多，顺便给你推荐灭蚊方法。

知识库搭好，下一步就是给 Agent 加知识库工具了。

最后要说明的是

目前我们的 Agent 只能从知识库中查询信息。

如果 Agent 能在干活的时候，自己把经验总结一下，存到知识库里面，那才叫牛逼。

这样我们的 Agent 就具备了自我学习和进化的能力，

那时候，它就真的能越用越懂你，越用越聪明了。
