## 面试官反馈 & 建议（作者置顶评论）

### 凉的核心原因

| # | 维度 | 反馈 |
|---|------|------|
| 1 | **基础概念** | 不能停留在背八股，要清楚每个优化手段的**适用场景和决策依据**（比如为什么分库、怎么分、分完有什么问题） |
| 2 | **AI技术栈** | RAG相对偏旧，建议跟进前沿（**MCP、Agent框架、Function Calling**等） |
| 3 | **代码** | LeetCode 4 最优解可以做到 **O(log(m+n)) 时间、O(1) 空间**，作者用的新建数组 O(n) 空间，未达最优 |
| 4 | **项目深度** | 校企合作项目可以理解，但如果有**自己从零搭建的AI工具**会更加分 |

### 关键启示

1. **八股要"懂为什么"不是"会背什么"**: 每个技术方案要知道适用场景、决策依据、以及引入的新问题
2. **AI技术栈要更新**: RAG 已经算"旧"技术了，MCP / Agent框架 / Function Calling 才是当前热点
3. **算法要刷透最优解**: Hard题不能只做出来，要追求最优复杂度
4. **项目要有个人亮点**: 校企合作/培训班项目不够，需要有自己从零搭建的项目

## 经验总结

1. **题量大但浅**: 33题连环问，面试官只听不追问 → 每题回答时间很短，无法展示深度
2. **AI方向面经**: RAG评估指标（检索准确率、召回率、MRR；生成ROUGE/BLEU/人工评估）是必考点
3. **概念辨析**: RAG vs Agent vs Skill vs MCP 的区别是AI应用岗高频题
4. **算法**: 手撕 Hard 题（中位数）要求 O(log(m+n)) 二分法，实习面试也考Hard
5. **基础八股密集**: MySQL索引、HashMap、分布式理论全覆盖
