## 题解参考（深度版）

### MCP（Model Context Protocol）
- **是什么**：Anthropic 提出的开放协议，标准化 AI 模型与外部工具/数据源的交互方式
- **为什么需要**：传统 AI 工具集成混乱，每个工具自定义 API → MCP 统一了"工具发现→调用→结果返回"的流程
- **核心概念**：Host（AI应用）→ Client（MCP客户端）→ Server（工具提供方）

### A2A（Agent-to-Agent）
- **是什么**：Google 提出的 Agent 间通信协议，解决多 Agent 协作编排问题
- **实现方式**：Agent Card（能力声明）+ 任务导向通信（Task Oriented Communication）

### 接口慢排查（Java）
1. **APM监控**（Arthas/Skywalking）→ 看哪个环节耗时长
2. **慢SQL** → EXPLAIN 分析索引 + 慢查询日志
3. **Redis** → 缓存命中率、大Key、热Key
4. **代码层面** → 死循环、频繁GC、线程阻塞
5. **外部依赖** → 下游接口超时、连接池耗尽

### 慢SQL优化
1. EXPLAIN 看 type（ALL→全表扫）、rows、Extra（Using filesort）
2. 索引优化：联合索引最左前缀、覆盖索引、索引下推
3. 分页优化：子查询延迟关联 → 先查主键再关联
4. SQL改写：避免函数/隐式转换导致索引失效

### Redis 使用场景
- 缓存（热点数据）+ 分布式锁（Redisson）+ 计数器（原子自增）
- 注意：大Key拆分、热Key本地缓存、持久化策略
