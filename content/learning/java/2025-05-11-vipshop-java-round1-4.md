## 🍀 唯品会面试的3个特点

1. **"工程味"非常重，不喜欢背八股**：面试官很反感"只会背书背概念"，更看重你能不能把理论落到业务场景。
2. **一定要深挖项目，拷打真实经历**："介绍一下你做过的最复杂的项目，遇到的难点是什么？" — 这道题几乎必问。面试官要的不是"我做了什么"，而是"为什么这么做、有什么权衡、遇到了什么坑"。
3. **技术栈偏"自研中台"，不是纯Spring Cloud**：唯品会这套自研框架，面试时如果能聊出对服务治理的理解，会很加分。

## 🌴 唯品会面试轮次（一共4轮）

## 一面｜技术基础面（45分钟，视频）

面试官是个说话不啰嗦的技术老哥，开场直接上题。

### 问题摘录

1. **"ArrayList和LinkedList的区别是什么？各自适合什么场景？"**
   - ArrayList基于动态数组，适合随机访问；LinkedList基于链表，适合频繁插入删除。
   - 追问 "ArrayList扩容机制？初始容量是多少？" — 无参构造初始容量10，1.5倍扩容。
   - 追问 "插入数据时谁更快？" — 头部插入LinkedList快，尾部插入ArrayList不一定。

2. **"HashMap的put过程说一下？1.7和1.8的区别？"**
   - 计算hash→定位桶→判断是否为空→判断key是否相同→插入。
   - 追问 "多线程下会有什么问题？" — 1.7会死循环，1.8已修复。
   - 追问 "ConcurrentHashMap怎么保证线程安全？" — 1.7分段锁，1.8CAS+synchronized。

3. **"a++是线程安全的吗？"**
   - 不是。
   - 追问 "Java有哪些保证线程安全的方式？" — synchronized、Lock、Atomic类、volatile（仅保证可见性）。
   - 追问 "volatile和synchronized的区别？" — volatile轻量级、不阻塞、不保证原子性。

4. **"final、finally、finalize的区别？"**
   - final修饰类/方法/变量；finally异常处理中保证执行；finalize是Object的垃圾回收方法。

5. **"MySQL索引有哪些类型？联合索引(a,b,c)，只查b会走索引吗？"**
   - 普通索引、唯一索引、主键索引、联合索引、全文索引。
   - 追问 "用explain分析SQL时，type列有哪些值？哪个性能最好？" — const最好，其次eq_ref/ref/range/index/ALL。

6. **"Redis的分布式锁怎么实现？"**
   - SETNX + 过期时间 + Lua脚本保证原子性释放。
   - 追问 "Redlock了解吗？" — 知道，但一般单机Redis分布式锁够用。

7. **"Spring AOP的实现原理？用的是哪种代理方式？"**
   - JDK动态代理（有接口）和CGLIB（无接口）。
   - 追问 "Spring默认用哪种？" — 有接口用JDK，没有则CGLIB。

8. **"唯品会的微服务框架OSP你了解吗？"**
   - OSP是唯品会自研的服务治理框架，类似Dubbo。
   - 追问 "和Spring Cloud有什么区别？" — OSP更轻量、更适合唯品会业务场景。

**手撕：** LRU缓存（LeetCode 146）

**📝 感想：** 一面Java基础面很扎实，但不会刻意刁难。Volatile、HashMap、ArrayList、MySQL索引、AOP这些题都有提到，只要基础打得好，问题不大。

## 二面｜项目深挖 + 架构设计面（60分钟，视频）

面试官是个架构师，开场就是"介绍一下你最满意的项目"。

### 问题摘录

1. **"详细介绍一个你做过的最复杂的项目，你在其中负责什么？"**
   - 讲了一个高并发订单系统的重构。
   - 追问 "遇到了什么技术难点？怎么解决的？" — 按STAR法则回答。
   - 追问 "如果再让你做一次，你会怎么改进？" — 回归本质是考验复盘能力。

2. **"高并发场景下，如何保证接口的幂等性？"**
   - 唯一流水号+Redis幂等字典+数据库唯一索引。

3. **"唯品会的数仓服务平台Hera你了解吗？"**
   - Hera是唯品会自研的数仓服务平台，基于Alluxio缓存表同步，人群计算从30分钟降到几分钟。
   - 追问 "如果是你设计类似的平台，会考虑哪些架构问题？" — 多引擎适配、缓存策略、资源隔离。

4. **"SpringBoot的自动装配原理？"**
   - @SpringBootApplication→@EnableAutoConfiguration→META-INF/spring.factories。

5. **"唯品会的OSP微服务框架和服务网关Janus，你觉得解决的核心问题是什么？"**
   - 服务注册发现、负载均衡、统一鉴权。
   - 追问 "和网关层应该做哪些事情？" — 限流、熔断、鉴权、日志、协议转换。

**手撕：** 合并K个有序链表（小顶堆）

**📝 感想：** 二面核心是项目深挖 + 唯品会技术栈的提前了解。提前看唯品会技术博客收获很大，面试官听到Hera时眼前一亮。建议大家去搜"唯品会技术博客"，提前读几篇技术文章。

## 三面｜交叉面 + 业务理解面（45分钟，视频）

面试官是个业务架构师，考的是技术广度和唯品会业务理解。

### 问题摘录

1. **"聊聊唯品会的个性化推荐业务，如果你是技术负责人会怎么架构？"**
   - 从数据采集→特征工程→模型推理→AB实验讲了架构。

2. **"Redis和数据库数据一致性怎么保证？"**
   - 先更新DB再删缓存。

3. **"唯品会的分布式任务调度平台Saturn和xxl-job比有什么优势？"**
   - Saturn基于Elastic-Job改良，支持分片并发处理、全域统一配置监控。适合唯品会大促期间大量定时任务调度场景。

4. **"如果线上出现CPU飙高，怎么快速定位问题？"**
   - top -Hp抓高占用线程→jstack抓堆栈→定位代码。

5. **"为什么想来唯品会？"**
   - 唯品会作为头部特卖电商，技术挑战大（大促高并发、海量数据处理）。而且公司近年在AI和大数据上持续投入（自研Hera平台）。

**手撕：** 无重复字符的最长子串（LeetCode 3）

**📝 感想：** 三面会夹杂业务视角。建议提前了解唯品会技术体系（OSP、Hera、Saturn）和业务模式。

## 四面｜HR面（30分钟）

### 问答摘要

- 离职原因？
- 期望薪资？——唯品会薪资参考25K-40K·15薪，5年经验35K左右
- 加班接受程度？— 大促期间有加班，平时相对稳定
- 反问：团队业务规划？

**📝 感想：** HR面比较常规。唯品会广州总部的工作环境和福利在广州算第一梯队。

## 🌳 唯品会Java面试核心考点

1. **Java集合**：HashMap底层(1.7/1.8区别)、ArrayList/LinkedList区别、ConcurrentHashMap实现
2. **并发编程**：volatile与synchronized、线程池参数、a++线程安全、乐观锁/悲观锁
3. **MySQL**：B+树索引结构、联合索引最左匹配、explain执行计划分析、索引失效场景
4. **Spring**：AOP代理方式、SpringBoot自动装配、IOC原理
5. **Redis**：数据结构、分布式锁实现、缓存三兄弟
6. **唯品会技术栈**：OSP微服务架构、Saturn任务调度、Hera数仓平台、Mercury全链路监控
7. **算法**：LRU、合并有序链表、无重复子串、树遍历
