# 饿了么 Java 后端一面面经

> 来源：[小红书](http://xhslink.com/o/AcYsBSLjo9S)（OCR 提取） | 提取日期：2026-05-22
> 侧重分布式 + 微服务 + MQ（进阶工程场）
> 面试形式：线上面试 55min | 面试官：资深后端
> 考察核心：分布式问题、微服务、消息队列、业务解决方案

---

## 1. 自我介绍（3min）

突出分布式项目、微服务搭建、中间件落地经验。

## 2. Java 高阶基础

1. **泛型原理**、类型擦除机制
2. **双亲委派模型**原理、作用，如何自定义类加载器？
3. **CompletableFuture** 异步编程使用场景

## 3. 分布式（核心重点）

1. 分布式 **CAP、BASE 理论**，最终一致性理解
2. **分布式事务**解决方案：2PC、TCC、本地消息表、Seata AT 模式优缺点
3. **分布式锁**的几种实现方式（Redis、Zookeeper）
4. **服务注册与发现**原理，Nacos 和 Eureka 区别

## 4. 消息队列 MQ

1. **RabbitMQ / Kafka** 核心架构、区别及适用场景
2. MQ 如何保证**消息不丢失、不重复消费、顺序性**？
3. **外卖场景**：如何用 MQ 实现异步下单、订单超时取消？

## 5. 微服务治理

1. 微服务**熔断、降级、限流**的区别及落地场景
2. **Sentinel** 核心功能、限流规则
3. **服务调用链路追踪**原理

## 6. 项目深度拷打

1. 项目中如何解决分布式事务问题？
2. **秒杀 / 下单场景**如何防超卖？具体实现方案
3. 高并发场景下系统**性能瓶颈**如何定位和优化？

## 7. 手撕算法

1. 有效的括号
2. 最长无重复子串

## 8. 反问环节

- 团队微服务架构现状
- 分布式技术迭代方向

## 面试总结

> 偏向进阶工程能力，聚焦分布式、微服务、业务解决方案，是饿了么正式岗一面核心考察方向。
