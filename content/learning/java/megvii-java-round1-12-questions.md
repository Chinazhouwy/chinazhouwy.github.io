# 旷视Java开发岗面经 — 12道高频题

> 来源：小红书 | 岗位：Java开发

## 一、项目 & 基础（必问）

### 1. 项目性能优化怎么搞？瓶颈怎么找？

**答题思路：**
- 用 Redis 缓存热点数据
- MySQL 加索引、慢SQL优化
- **一定要说量化结果**（如：接口RT从500ms降到50ms）

### 2. 限流算法有哪些？怎么选？

| 算法 | 特点 | 适用场景 |
|------|------|----------|
| 计数器 | 简单，临界突变 | 简单限流 |
| 滑动窗口 | 平滑 | 较精确控制 |
| 令牌桶 | 允许突发 | **适合突发流量** |
| 漏桶 | 平滑输出 | 保护下游 |

### 3. Redis 是单线程吗？高性能靠啥？

- **核心网络模型是单线程**（IO多路复用）
- 高性能原因：纯内存、单线程无锁竞争、数据结构优化、IO多路复用

## 二、Redis 高频场景题（重灾区）

### 4. 高并发热点 key 怎么处理？

- **多级缓存**：本地缓存 + Redis
- **key 拆分**：将热点 key 拆为多个 key 分散压力
- **热点 key 预热**：系统启动时提前加载到缓存
- **别只说一个方法**，要列出完整方案组合

### 5. Redis 持久化磁盘IO瓶颈怎么破？

- 调整 AOF 刷盘策略（`everysec`）
- RDB + AOF 结合使用
- 用大内存机器
- **主从分离**：从节点负责持久化

### 6. Redis 集群节点挂了怎么办？

排查步骤：
1. 查看哨兵/集群状态
2. 排查日志
3. 主从切换
4. 故障节点恢复

## 三、MySQL 性能优化

### 7. 查询慢怎么排查优化？

- `EXPLAIN` 看执行计划（type/rows/Extra）
- 加索引（覆盖索引、联合索引）
- 优化 SQL 语句
- 分库分表
- 读写分离

### 8. 除了索引，慢SQL还有啥优化方法？

- 避免 `SELECT *`
- `LIMIT` 优化（深分页 → 子查询延迟关联）
- 大表分页优化
- 批量操作（batch insert/update）
- 事务控制（缩小事务范围）

## 四、数据结构 & JVM

### 9. ArrayList 和 LinkedList 怎么选？

| 操作 | ArrayList | LinkedList |
|------|-----------|------------|
| 查询 | ✅ O(1) | ❌ O(n) |
| 尾部插入 | ✅ O(1)（均摊） | ✅ O(1) |
| 中间插入 | ❌ O(n) | ✅ O(n) |
| 内存 | 连续内存 | 分散节点+指针 |

**结论**：查多用 ArrayList，增删多用 LinkedList

### 10. 频繁增删用啥集合？为啥？

**LinkedList**（双向链表），因为 ArrayList 增删要搬运元素

### 11. Minor GC 和 Full GC 啥区别？啥时候触发？

| GC类型 | 范围 | 速度 | 触发时机 |
|--------|------|------|----------|
| Minor GC | 年轻代 | 快 | Eden区满 |
| Full GC | 整个堆+方法区 | 慢 | 老年代满、System.gc()、晋升失败、Metaspace满 |

### 12. JVM 年轻代分几个区？作用是啥？

**Eden + 2个 Survivor 区（S0/S1）**

复制算法流程：
1. 新对象 → Eden
2. Minor GC：Eden+S0 存活对象 → S1
3. 年龄+1，交换 S0/S1 角色
4. 年龄 > 阈值（默认15）→ 老年代
