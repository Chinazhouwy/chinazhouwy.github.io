## 二、后端八股合集

### 操作系统 & 计算机网络
1. 项目里用 Python 多进程解决 OOM，放弃多线程的原因
2. 从操作系统层面，进程和线程核心差异在哪
3. FastAPI 项目延迟很低，客户端发起请求，完整 TCP 三次握手过程
4. 实时消息推送选用 WebSocket，舍弃 HTTP 轮询的理由

### JVM 相关
1. Java 运行时 JVM 内存分区划分
2. 堆中的对象百分百会被 GC 回收吗
3. 不同引用类型的回收规则（强/软/弱/虚）
4. 详细介绍垃圾回收整体流程
5. 标记清除产生大量内存碎片，后续申请超大内存如何处理

### Redis
1. 布隆过滤器、互斥锁、逻辑过期各自落地解决什么问题
2. 物理过期、逻辑过期两者区别
3. HyperLogLog、ZSet、Bitmap 底层原理 + 适用场景
4. 场景题：统计连续 7 天每日都上线的活跃用户交集怎么实现
5. Redis 基础数据结构全梳理，ZSet 依托跳表的实现原理
6. 验证码来源、Token 存储位置
7. 商户修改业务信息，库 + 缓存双更新流程
8. 更新 DB 删缓存，如何保证最终数据一致性

### RabbitMQ
1. 整条链路怎么确保消息完整落地、百分百入库，梳理生产到消费全流程
2. 死信队列日常业务处理方案
3. 多种手段实现消息消费幂等

### MySQL
1. 用户表 + 签到表结构，按月查单人签到记录，索引优化设计
2. 不借助 Redis，纯 MySQL 方案防止高并发重复点赞
3. 高并发场景悲观锁、乐观锁区别，悲观锁落地弊端

### 语言底层拓展
1. Java、Python、Go 三门语言对比差异
2. 编译型语言和解释型语言本质区别
3. Go 的 GMP 调度模型详解

## 个人小结

同样是三合一，和之前收集的字节 AI 后端考察逻辑很像，知识点杂、覆盖面广。

### 覆盖知识点
- **操作系统**: 进程vs线程、TCP三次握手、WebSocket
- **JVM**: 内存分区、GC流程、引用类型、内存碎片
- **Redis**: 布隆过滤器/互斥锁/逻辑过期、HyperLogLog/ZSet/Bitmap、缓存一致性
- **RabbitMQ**: 消息可靠投递、死信队列、幂等消费
- **MySQL**: 索引设计、乐观锁vs悲观锁
- **语言**: Java/Python/Go对比、编译vs解释、Go GMP
- **算法**: LRU缓存、合并K有序链表
- **AI**: RAG流程、向量检索、Agent架构设计
