# 腾讯瑞驰 后端 Java 二面面经（偏 JVM + 性能调优）

> 来源：小红书面经 | 标签：Java、JVM、性能调优、分布式

## 面试概况

- **岗位**: 腾讯瑞驰 - 后端 Java 开发（二面）
- **特点**: 偏 JVM + 性能调优，总时长约 60 分钟
- **形式**: 7 大模块，含手写算法

---

## 一、项目深挖（15min）

1. **性能优化项目**：介绍一个你做过的性能优化项目，优化前指标、优化手段、优化后指标
2. **JVM 参数配置**：项目中 JVM 参数怎么配置？GC 情况如何？有没有 GC 频繁 / Full GC 问题？
3. **应用监控**：怎么监控 Java 应用（CPU / 内存 / GC / 接口耗时 / 异常）？用了哪些工具？
4. **多线程场景**：项目中线程安全、锁竞争、死锁，怎么处理？

## 二、Java 核心 + JVM（15min）

5. **JMM 内存模型**：主内存 / 工作内存、可见性、原子性、有序性，volatile 如何保证？
6. **锁机制**：synchronized 与 Lock 区别，AQS 原理（独占 / 共享、队列、CAS）
7. **类加载过程**：加载 → 验证 → 准备 → 解析 → 初始化，双亲委派模型及破坏场景
8. **内存问题**：内存泄漏 vs 内存溢出，常见内存泄漏场景（静态集合、ThreadLocal、未关闭流）及排查

## 三、中间件 & 分布式（10min）

9. **Redis 持久化**：RDB / AOF，各自优缺点、混合持久化、恢复流程
10. **Redis 内存优化**：过期策略、内存淘汰、大 Key 拆分、对象压缩
11. **分布式事务**：CAP 理论、BASE 理论、2PC / 3PC 问题、Seata AT 模式原理

## 四、数据库（5min）

12. **MySQL 主从复制**：binlog / relaylog 原理，主从延迟原因及优化
13. **分库分表**：水平 / 垂直拆分、分片键选择、路由规则、跨库查询

## 五、算法（5min，手写）

14. **手写快排**：并分析时间复杂度、稳定性
15. **二叉树后序遍历**：迭代写法（非递归）

## 六、系统设计（5min）

16. **设计一个高可用限流网关**：
    - 限流算法：计数器 / 滑动窗口 / 令牌桶 / 漏桶
    - 集群限流
    - 降级策略

## 七、行为问题（5min）

17. 遇到技术瓶颈怎么解决？举个印象最深的例子
18. 对加班、压力怎么看？能接受高强度工作吗？

---

## 经验总结

1. **JVM 是核心考点**：二面 15min 全在问 JVM（JMM、AQS、类加载、内存泄漏），必须深入
2. **性能优化要能说清指标**：优化前 → 手段 → 优化后，有数据支撑
3. **监控工具要会用**：jstat、jmap、Arthas 是标配
4. **算法手写基础题**：快排 + 二叉树遍历（迭代），难度不高但要求手写
5. **系统设计聚焦限流**：限流网关是经典题，四种算法 + 集群 + 降级都要掌握
6. **时间分配紧凑**：60min 覆盖 7 大模块，每模块 5-15min，回答要精炼
