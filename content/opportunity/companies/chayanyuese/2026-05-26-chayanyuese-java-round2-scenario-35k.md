# 茶颜悦色Java开发二面 — 场景题实战（35k offer）

> 来源：小红书 | 岗位：Java后端开发 | 经验：2年电商后端，双非一本，日均订单破百万

## 面试题

### 项目引入 + 秒杀系统设计

**1. 设计一个爆款饮品秒杀系统**

"幽兰拿铁"上新时瞬时10万QPS，从缓存、限流、库存扣减到订单创建的全流程设计。

**核心要点：**
- **缓存**：Redis 预热商品库存 + 热点 key 拆分
- **限流**：令牌桶（漏桶也可以），用户维度的限流
- **库存扣减**：Redis `DECR` 原子扣减 + 异步落库
- **订单创建**：MQ 削峰，异步处理
- **兜底**：降级、熔断

### 2. 订单超时未支付自动取消

如何实现高精度（30分钟精确到秒）且低延迟的订单状态回滚？

**方案：**
- Redis 延迟队列（`ZSET` 存储过期时间戳 + 订单ID）
- 定时任务轮询 + Redisson `RDelayedQueue`
- RocketMQ 延迟消息（18个等级）

### 3. 多门店库存同步问题

某款原料缺货时，如何避免跨门店订单分配导致的超卖？

**方案：**
- 分布式锁（按门店维度加锁）
- 本地库存预热 + Redis 实时扣减
- 最终一致性 + 补偿任务

### 4. 会员积分并发累加

高并发下积分更新原子性？对比 Redis INCR 与数据库乐观锁。

| 方案 | 优势 | 劣势 |
|------|------|------|
| Redis INCR | 原子性、高性能 | 可能丢数据（需持久化） |
| 数据库乐观锁 | 可靠、事务保障 | 性能差、冲突回滚 |

**推荐**：Redis INCR 实时累计 + 定时同步到 DB

### 5. 设计一个动态定价系统

根据天气、时段、库存自动调整饮品价格，如何避免频繁DB查询？

**方案：**
- 定价规则缓存到 Redis（Hash 结构）
- 价格计算本地化（规则引擎）
- 天气/时段数据异步更新缓存

### 6. 分布式事务实战

用户下单同时使用优惠券和积分，如何保证两者同时成功或失败？

**方案：**
- **TCC**（Try-Confirm-Cancel）：积分冻结 + 优惠券锁定
- **本地消息表**：下单事件 + 异步消费
- **Seata AT** 事务模式
