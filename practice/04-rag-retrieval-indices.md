## 第 4 题 · （待补充）

**题目**：RAG 系统中的检索索引有哪些类型？稠密向量索引和稀疏向量索引分别适合什么场景？生产环境中通常会怎么做检索？为什么单独用一种索引不够？

### 用户回答（摘要）

> 完全不会，只知道一个余弦向量检索。

### 评分：1/10

### 扣分点

1. 稠密/稀疏概念完全不知道（-4）
2. BM25、倒排索引未掌握（-2）
3. 混合检索（Hybrid Search）未了解（-2）
4. RRF 融合算法未了解（-1）

### 正确答案要点

| 类型 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 稠密向量（Dense） | Embedding + 余弦相似度 | 语义理解 | 精确匹配差 |
| 稀疏向量（Sparse） | BM25 / TF-IDF | 精确匹配 | 不理解语义 |
| 混合检索 | 两者 + RRF 融合 | 效果最好 | 稍慢 |

- RRF = Reciprocal Rank Fusion，公式 score=1/(k+rank)，纯数学融合
- Reranker = 小模型打分，输入(问题, 文档)，输出相关性分数
- 生产标配：稀疏+稠密双路检索 → RRF/Reranker 融合 → Top K 喂 LLM
