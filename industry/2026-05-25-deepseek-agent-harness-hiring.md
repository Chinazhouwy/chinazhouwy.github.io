# DeepSeek 招聘 — Agent Harness 岗位解读

**来源：** 小红书  
**时间：** 2026-05-25  
**标签：** #DeepSeek #Agent #Harness #招聘 #行业趋势

---

## 核心洞察

DeepSeek 挂出两个新岗位：**Agent Harness 研发工程师** + **Agent Harness 产品经理**，坐标北京。

> **关键词：Harness** — 模型本身只是能力内核，把模型能力变成真正能用的 Agent 产品，中间所有的工程、产品、交互设计工作，统称为 **Harness**。

这说明 DeepSeek 已经在**系统性地思考 Agent 产品化**这件事了。

---

## 岗位 JD 详情

### 1. Agent Harness 研发工程师

**团队使命：** `Model + Harness = Agent`

**主要职责：**
- 参与设计 Harness 产品技术架构与选型
- 参与开发 Harness 产品
- 与研究员紧密合作，定义和实现 Harness 领域前沿创新
- 与模型训练团队深度合作，实现模型与 Harness 的共同进化与深度适配
- 以内部真实任务作为产品和模型训练的重要反馈源
- 分析用户反馈，协助维护产品用户社群
- 协助项目管理

**任职要求：**
- 2年+ 软件开发经验（优秀可放宽）
- 知名高校本科+
- **熟练使用 AI Agent 工具进行软件开发** — 在 AI 辅助下，能在无直接经验的领域（语言/技术/框架）做有质量保证的编程
- **Agent 产品高强度用户** — 对 Agent Harness 有极大热情，对模型行为有品味有判断力
- 熟悉 LLM/Agent 机制：LLM API、KV Cache、Agent Loop、Tool Use、Reasoning、Planning、**Skills、MCP、Memory、Subagent、Multi-Agent**
- 对 Prompt Engineering、Context Engineering、Harness Engineering 有深入理解
- UI/UX 设计素养，能在 AI 辅助下完成原型/UI 设计
- 英文沟通能力

**加分项：**
- AI 行业经验、AI 产品经验
- 小团队研发工程师快速迭代经验
- 个人开源作品及开源社区深度贡献
- 与研究员协作经验或科研经验
- 有含金量的比赛奖项或论文发表
- 其它超乎常人的才能

---

### 2. Agent Harness 产品经理

**核心看点：** 要求中直接点名了一串竞品 — **Claude Code、Codex、Cursor、Manus、OpenClaw**……要求候选人**深度使用过这些产品并融入到日常工作生活中**。

这基本上把当前 Agent 赛道的主要玩家全部列了一遍，DeepSeek 显然对这个市场做了非常细致的研究。

（注：OpenClaw 是该平台所使用的底层系统 —— 面经笔记中 Agent 岗位的要求与 DeepSeek JD 高度重合）

## 深度分析

### 为什么这对你很重要

DeepSeek **正式下场做 Agent 产品**了。考虑到他们在模型层的技术储备，加上这种**模型 + Harness 联合进化**的打法，2026 下半年的 Agent 赛道会比现在卷得多。

### JD = 学习路线图

光是 JD 的任职要求，本身就是一份很好的 **Agent 学习路线图**：

| 技术要求 | 对应学习内容 |
|---------|------------|
| Agent Loop + Tool Use | ReAct 框架、Function Call |
| Reasoning + Planning | 思维链、规划算法 |
| **Skills** | 技能系统设计与热插拔 |
| **MCP** | Model Context Protocol 协议 |
| **Memory** | 短期/长期记忆、压缩策略 |
| **Subagent** | 子代理编排与任务分解 |
| **Multi-Agent** | 多 Agent 协作、A2A 协议 |
| Prompt Engineering | 提示词工程 |
| Context Engineering | 上下文管理与窗口优化 |
| Harness Engineering | Agent 产品化工程 |

### 启示

1. DeepSeek 的 Agent 产品以**桌面端**为主（JD 提到"桌面端 Agent 产品"）
2. 强调 **AI 辅助开发能力** — 会用 AI 工具写代码本身成了硬性要求
3. **模型 + Harness 深度耦合** — 不只是 API 调用，要深度参与模型训练反馈
4. 产品经理也要**深度使用竞品** — 说明他们很务实，不走纯概念路线
